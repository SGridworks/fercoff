## The Democratization Problem

### The Current State: Synthetic Data Only Gets You So Far

SGRIDWORKS gave you synthetic datasets and trained models. Your engineers have been experimenting — testing anomaly detection, building predictive maintenance classifiers, prototyping natural language interfaces to SCADA data.

**What they've learned:** The models work. The approach is sound. The potential is real.

**What they can't do yet:** Run any of it on real operational data.

### Why Live Data Stays Locked Away

| Barrier | Why It Blocks You |
|---------|-------------------|
| **CEII Classification** | Your grid topology, real-time SCADA, and operational parameters are Critical Energy Infrastructure Information. One leak is a reportable incident [1]. |
| **Cloud API Prohibition** | Sending CEII to OpenAI, Anthropic, or any cloud LLM violates 18 CFR 388.113 and potentially FPA Section 215A [2][3]. |
| **Procurement Complexity** | Traditional air-gapped AI infrastructure requires $2M+ data center investments and 6-12 month procurement cycles. |
| **Compliance Uncertainty** | Even if you buy the hardware, NERC CIP compliance for AI systems is poorly understood. Nobody wants to be the test case. |

**Result:** Your most capable engineers are stuck analyzing synthetic data while operational insights remain buried in logs they can't touch.

### The Hidden Cost

Every day your engineers can't experiment on live data:
- Anomaly detection models miss real failure patterns
- Predictive maintenance stays theoretical
- Operational knowledge walks out the door with retiring staff
- Your grid runs less efficiently than it could

**The question isn't whether to unlock live data. It's how to do it without creating compliance exposure.**

## The Sandbox Architecture

### The Core Insight

You don't need a data center. You need two Mac Minis and a network configuration that keeps CEII physically isolated from external connections.

**The SGRIDWORKS Sandbox:**

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  YOUR ENGINEER'S LAPTOP (Anywhere)                                          │
│  • Writes Python, Jupyter notebooks                                          │
│  • Submits jobs via Tailscale VPN                                            │
│  • Never touches CEII directly                                               │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    │ Encrypted Tailscale tunnel
                                    │ (metadata only, no data flows)
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  MAC MINI A: GATEWAY (Office/Control Room)                                  │
│  • Job orchestration and queueing                                            │
│  • Code validation and sandboxing                                            │
│  • Results aggregation (non-sensitive only)                                  │
│  • Cloud API access for non-CEII tasks (optional)                            │
│                                                                              │
│  Models: Qwen 2.5-3B (local), Kimi K2.5 (cloud, non-CEII only)              │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    │ Air-gapped transfer (manual or data diode)
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  MAC MINI B: SECURE SANDBOX (Substation/Control Room)                       │
│  • Air-gapped — no external network                                          │
│  • CEII data at rest, encrypted                                              │
│  • Qwen 2.5-3B/7B inference on live data                                     │
│  • Immutable audit logging                                                   │
│                                                                              │
│  Data Sources: SCADA historian, maintenance logs, AMI data (your actual data)│
└─────────────────────────────────────────────────────────────────────────────┘
```

### What Makes This a "Sandbox"

| Sandbox Property | Implementation |
|------------------|----------------|
| **Isolation** | Mini B has no route to external networks. CEII cannot leave. |
| **Observability** | Every query, every result, every model output is logged. |
| **Reversibility** | Experiments run on copies. Original data is never modified. |
| **Bounded Risk** | Worst case: sandbox is compromised. CEII stays contained. |
| **Auditability** | Complete trail for compliance review. |

### Physical Setup

**Mini A (Gateway):**
- Location: Office, control room, or anywhere with internet
- Network: Ethernet to your LAN, WiFi backup
- Access: Your engineers SSH/Tailscale in
- Security: Standard office physical security

**Mini B (Secure Sandbox):**
- Location: Substation, control room, or locked equipment closet
- Network: Ethernet to your OT network (SCADA historian, etc.)
- **No WiFi. No internet. Air-gapped.**
- Physical security: Locked cabinet, access log, tamper-evident seals
- Environmental: 0-35°C operating range [4]. Climate-controlled preferred.

### Software Installation

**On Mini A (Gateway) — with internet:**

```bash
# Install Homebrew
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install dependencies
brew install node@22 tailscale ollama

# Install OpenClaw
npm install -g openclaw

# Configure Tailscale
sudo tailscale up

# Pull Qwen models (for non-CEII tasks)
ollama pull qwen2.5:3b

# Start OpenClaw gateway
openclaw gateway setup
openclaw gateway start
```

**On Mini B (Sandbox) — air-gapped:**

```bash
# Install from USB (pre-downloaded packages)
# See Appendix A for offline installation guide

# Configure Tailscale (no internet, mesh only)
sudo tailscale up --accept-routes=false

# Pull Qwen models (from USB transfer)
ollama pull qwen2.5:3b  # 2GB
ollama pull qwen2.5:7b  # 4.5GB

# Enable FileVault (full disk encryption)
sudo fdesetup enable

# Disable unnecessary services
sudo launchctl unload /System/Library/LaunchDaemons/com.apple.mDNSResponder.plist
sudo launchctl unload /System/Library/LaunchDaemons/com.apple.awacsd.plist
```

### Network Configuration

**Tailscale ACLs (on Mini A):**

```json
{
  "acls": [
    {"action": "accept", "src": ["group:engineers"], "dst": ["mini-a:*"]},
    {"action": "accept", "src": ["mini-a"], "dst": ["mini-b:11434"]},
    {"action": "deny", "src": ["*"], "dst": ["mini-b:*"]}
  ],
  "tagOwners": {
    "mini-a": ["admin"],
    "mini-b": ["mini-a"]
  }
}
```

**Firewall on Mini B (pf):**

```bash
# Block all outbound
block out all

# Allow Tailscale mesh only (no exit node)
pass in on tailscale0
pass out on tailscale0

# Block DNS, NTP, everything else
block out proto udp
block out proto tcp
```

### Data Ingestion

**Option 1: USB Transfer (Small Datasets < 10GB)**

1. Export from OT historian to encrypted USB
2. Walk USB to Mini B
3. Import, decrypt, process
4. Results to separate encrypted USB for export

**Option 2: Dedicated Ethernet (Large Datasets / Real-time)**

1. Dedicated Ethernet cable from OT historian to Mini B
2. No switch, no router, point-to-point
3. Data diode or manually controlled disconnect
4. Automated or scheduled batch transfer

**Option 3: Staging Server (Advanced)**

1. Staging server on OT network
2. One-way sync to Mini B via rsync over SSH
3. Automated validation and sanitization
4. Manual approval for sensitive queries

## Operating Procedures

### Daily Operations

**Your Engineer:**
1. SSH to Mini A (Gateway) via Tailscale
2. Write Python/Jupyter notebook as usual
3. Submit job to sandbox queue
4. Wait for results (typically seconds to minutes)
5. Iterate, refine, experiment

**What Happens Automatically:**
- Code validated and sandboxed
- Query logged with timestamp, user, model version
- Inference runs on Mini B (air-gapped)
- Results returned (non-sensitive aggregation only)
- Audit trail updated

### Model Updates

**Updating Qwen on Air-Gapped Mini B:**

```bash
# On Mini A (with internet)
ollama pull qwen2.5:7b
ollama save qwen2.5:7b -o qwen2.5-7b-new.bin
gpg --armor --detach-sign qwen2.5-7b-new.bin

# Transfer to USB
cp qwen2.5-7b-new.bin* /Volumes/USB/

# Walk USB to Mini B
# On Mini B (air-gapped)
gpg --verify qwen2.5-7b-new.bin.asc
ollama load qwen2.5-7b-new.bin
ollama run qwen2.5:7b "Test query"
```

### Backup and Recovery

| Data | Frequency | Method | Location |
|------|-----------|--------|----------|
| Model weights | Weekly | USB copy | Mini A + offsite |
| Configuration | Daily | Git | Mini A + GitHub |
| Audit logs | Continuous | Append-only | Mini B (local) |
| CEII data | Per-job | Job-scoped | Mini B only |

**Disaster Recovery (Both Minis Fail):**

1. Deploy spare Mac Mini from secure storage
2. Restore configuration from Git
3. Re-install models from USB backup
4. Resume operations within 2 hours
